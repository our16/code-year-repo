## 链接格式规范

### ✅ 推荐格式

```markdown
# 短链接 + 文件名
[并发扫描](仓库并发扫描功能说明.md)

# 带描述
[并发扫描](仓库并发扫描功能说明.md) - 多项目并行采集

# 描述性链接
[Git并发扫描功能](仓库并发扫描功能说明.md)
```

### ❌ 不推荐格式

```markdown
# 带路径前缀
[并发扫描](md/仓库并发扫描功能说明.md)  ❌
[并发扫描](./仓库并发扫描功能说明.md)  ❌

# 文件名作为链接文字
[仓库并发扫描功能说明.md](仓库并发扫描功能说明.md)  ❌
```

## 维护建议

### 添加新文档

1. 在 `md/` 目录创建 `.md` 文件
2. 使用规范的链接格式引用其他文档
3. 不在 `md/` 目录存放代码或配置文件

### 优化现有文档

**方法1: VSCode正则替换**

```
查找: \[([^\]]+)\]\(\.?/?md/([^)]+)\)
替换: [$1]($2)
```

**方法2: 手动优化**

1. 去掉路径前缀（`md/`、`./`、`../md/`）
2. 缩短链接文字
3. 添加描述性说明

### 目录结构规范

```
md/                          # 只存放文档
├── *.md                     # Markdown文档文件
├── PROJECT_STRUCTURE.md
├── 链接格式优化说明.md
└── ...

src/                         # 代码文件
├── *.py
└── ...

config/                      # 配置文件
├── *.yaml
├── *.json
└── ...
```

## 验证检查

### 检查命令

```bash
# 检查是否有代码文件
find md -name "*.py"
find md -name "*.yaml"
find md -name ".link_map*"

# 检查链接格式
grep -r '\[.*\](md/' md/*.md
grep -r '\[.*\](\./md/' md/*.md
```
